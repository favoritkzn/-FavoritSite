## 1. Философия продукта

**Фаворит — это не CRM. Это цифровой дом академии.**

Мы создаём продукт, который ощущается как современное спортивное приложение — такое, которым родители *хотят* пользоваться, а не *вынуждены*. Каждый экран должен передавать энергию футбола, заботу о детях и профессионализм академии.

### Ключевые принципы

| Принцип | Смысл |
|---------|-------|
| **Радость, а не отчётность** | Родитель открывает приложение и видит прогресс ребёнка, а не таблицу с цифрами |
| **Простота превыше функций** | Лучше сделать 10 вещей идеально, чем 50 посредственно |
| **Контекст вместо навигации** | Вся важная информация — на главном экране, без 5 кликов вглубь |
| **Живой клуб** | Фото с тренировки, гол вчера, ближайший матч — академия «дышит» через экран |
| **Уважение к времени** | Тренер отмечает посещаемость за 30 секунд. Родитель оплачивает абонемент за 2 клика |

### Миссия

Связать академию, тренеров и семьи в единую цифровую экосистему — чтобы каждый участник чувствовал себя частью команды «Фаворит».

### Видение

Стать эталоном цифровизации детских футбольных академий в России: продукт, на который ссылаются конкуренты.

## 2. Цели платформы

### Бизнес-цели

1. **Заменить устаревший сайт** — современный имидж академии уровня крупных клубов
2. **Снизить нагрузку на администратора** — автоматизация абонементов, оплат, уведомлений
3. **Частично заменить «ТСК Сотрудник»** — единая точка для родителей и тренеров
4. **Увеличить вовлечённость родителей** — ежедневное использование вместо раз в месяц
5. **Монетизация через магазин** — форма, экипировка, мерч с конструктором

### Продуктовые цели

1. Родитель узнаёт о тренировке, оплате и прогрессе ребёнка **без звонка в офис**
2. Тренер ведёт группу **с телефона за минуты**, а не за часы
3. Директор видит состояние академии **в одном экране**, без Excel
4. Гость (новый клиент) понимает ценность академии **за 30 секунд** на главной

### Метрики успеха (MVP + 3 месяца)

| Метрика | Цель |
|---------|------|
| DAU родителей / MAU | ≥ 40% |
| Время отметки посещаемости (тренер) | < 1 мин на группу |
| Онлайн-оплата абонементов | ≥ 70% |
| NPS родителей | ≥ 50 |
| Обращения в офис по рутинным вопросам | −50% |

## 4. Основные пользовательские сценарии

### Родитель

#### Сценарий 1: Утро понедельника
> Анна открывает приложение за завтраком. На Dashboard видит: сегодня тренировка в 17:00, осталось 8 занятий по абонементу, вчера тренер загрузил 12 фото. Один тап — смотрит альбом.

#### Сценарий 2: Оплата абонемента
> За 3 дня до окончания абонемента — push-уведомление. Анна нажимает «Оплатить» прямо с Dashboard. ЮKassa, 2 клика, готово. Автопродление включено.

#### Сценарий 3: Прогресс ребёнка
> После турнира Дмитрий открывает карточку сына: 2 гола, 1 передача, 5 матчей в сезоне. Делится скриншотом в семейный чат.

#### Сценарий 4: Покупка формы
> Новый сезон — нужна форма. Конструктор: вводит фамилию «Иванов», номер 10. Видит превью в реальном времени. Добавляет в корзину, оформляет.

### Тренер

#### Сценарий 5: Отметка посещаемости
> После тренировки Игорь открывает телефон → Dashboard → «Отметить посещаемость» → список группы с тумблерами → Готово. 30 секунд.

#### Сценарий 6: Результат матча
> Команда выиграла 3:1. Игорь добавляет матч: счёт, голы (кто забил), передачи. Родители видят в статистике детей.

#### Сценарий 7: Объявление группе
> Отмена тренировки из-за погоды. Игорь пишет объявление — push всем родителям группы.

### Администратор / Директор

#### Сценарий 8: Утренний обзор
> Елена открывает Admin Dashboard: 52 ребёнка, 3 абонемента истекают на этой неделе, 2 неоплаченных, посещаемость за месяц 87%. Без Excel.

#### Сценарий 9: Новый ученик
> Родитель звонит. Елена создаёт ребёнка, привязывает к группе и родителю. Родитель получает приглашение в личный кабинет.

### Гость

#### Сценарий 10: Первое знакомство
> Мама ищет «футбольная школа Казань». Попадает на сайт Фаворит: красивый hero, тренеры, расписание, цены. Записывается через форму контактов.

## 5. Почему родители должны пользоваться сайтом ежедневно

Обычный сайт академии посещают раз в месяц — чтобы узнать расписание или оплатить. **Фаворит создаёт привычку.**

### Ежедневные триггеры

| Триггер | Механизм | Эмоция |
|---------|----------|--------|
| **Расписание** | «Сегодня тренировка в 17:00» на Dashboard | Подготовка |
| **Фото** | Тренер загрузил фото — push «12 новых фото с тренировки» | Радость, гордость |
| **Прогресс** | «Артём: +1 гол на этой неделе» | Гордость за ребёнка |
| **Абонемент** | Прогресс-бар «осталось 5 тренировок» | Контроль |
| **Новости** | «Фаворит выиграл турнир!» | Принадлежность к клубу |
| **Объявления** | «Завтра сбор в 8:00» | Не пропустить важное |

### Привычка (Hook Model)

```
Триггер (push/расписание) → Действие (открыть Dashboard) →
Награда (фото, прогресс, новость) → Инвестиция (профиль ребёнка, история)
```

Родитель вкладывает данные (фото ребёнка, настройки) → чем больше истории, тем ценнее продукт → выше retention.

## 6. Ценность для каждой роли

### Для директора

| Было | Стало |
|------|-------|
| Excel-таблицы посещаемости | Автоматическая аналитика на Dashboard |
| Звонки «когда тренировка?» | Родители видят расписание сами |
| Ручные напоминания об оплате | Автоуведомления + онлайн-оплата |
| Разрозненные чаты с тренерами | Единая система объявлений |
| Нет понимания загрузки групп | Визуальная статистика по группам |

**Главная ценность:** прозрачность и контроль без микроменеджмента. 5 минут утром — и директор знает состояние академии.

### Для тренеров

| Было | Стало |
|------|-------|
| Бумажный журнал посещаемости | Отметка с телефона за 30 сек |
| Устные объявления родителям | Push-уведомления группе |
| Нет единого места для результатов | Матчи, голы, передачи в системе |
| Родители звонят лично | Вся информация в кабинете |

**Главная ценность:** меньше рутины — больше тренировок. Тренер тренирует, а не администрирует.

### Для родителей

| Было | Стало |
|------|-------|
| Звонить в офис | Всё в личном кабинете 24/7 |
| Не знать, ходит ли ребёнок | История посещений в реальном времени |
| Не видеть прогресс | Статистика: голы, матчи, посещаемость |
| Оплата только в офисе | Онлайн-оплата за 2 клика |
| Пропускать фото с тренировок | Push + альбомы в кабинете |

**Главная ценность:** быть рядом с ребёнком в его спортивном пути — даже когда не на поле.

## 7. Чем система отличается от обычной CRM

| Критерий | Классическая CRM | Фаворит |
|----------|------------------|---------|
| **Ощущение** | Офисная программа, таблицы, серость | Спортивное приложение, энергия, красный акцент |
| **Главный экран** | Таблица записей | Dashboard с карточками, фото, прогрессом |
| **Навигация** | 10+ пунктов меню | 5-6 ключевых разделов, остальное — с Dashboard |
| **Данные** | Строки и столбцы | Карточки, прогресс-бары, аватары, таймлайны |
| **Действия** | «Создать запись» → форма из 15 полей | Быстрые действия: «Оплатить», «Отметить», «Загрузить фото» |
| **Уведомления** | Email-дайджест раз в неделю | Push в момент события: фото, гол, напоминание |
| **Мобильность** | Адаптивная таблица (плохо) | Mobile-first, touch-friendly |
| **Эмоции** | Нейтрально-деловые | Гордость за ребёнка, радость от фото, азарт статистики |
| **Онбординг** | Инструкция на 10 страниц | Интуитивно: «Вот ваш ребёнок, вот расписание» |
| **Визуал** | Синий/серый корпоратив | Красный/белый спортивный, как Nike/Adidas digital |

### Формула

```
CRM = управление данными
Фаворит = управление опытом семьи в академии
```

## 8. Emotional Design

> Подробные визуальные принципы — в [UI_GUIDELINES.md](./UI_GUIDELINES.md). UX-паттерны — в [UX_PRINCIPLES.md](./UX_PRINCIPLES.md).

### Принцип

**Каждый пиксель должен говорить: «Это футбол. Это твой клуб. Это современно».**

### Эмоциональная карта экранов

| Экран | Целевая эмоция | Как достигаем |
|-------|----------------|---------------|
| Публичная главная | Вдохновение, доверие | Hero с детьми на поле, красный акцент, социальное доказательство |
| Dashboard родителя | Спокойствие, гордость | Фото ребёнка, прогресс, ближайшая тренировка |
| Карточка ребёнка | Гордость | Статистика как в спортивном приложении |
| Посещаемость | Прозрачность | Визуальный календарь, не таблица |
| Оплата | Уверенность | Чистая форма, success-анимация |
| Dashboard тренера | Эффективность | Быстрые действия, минимум шума |
| Dashboard админа | Контроль | Крупные метрики, тренды, без перегруза |
| 404 | Лёгкость | Иллюстрация с мячом, юмор |

### Что НЕ делать

- ❌ Таблицы на 15 колонок без необходимости
- ❌ Серые формы с 20 полями
- ❌ Терминология CRM: «сущность», «запись», «воронка»
- ❌ Мелкий текст, плотная вёрстка
- ❌ Иконки без подписей в ключевых местах
- ❌ Пустые экраны без иллюстраций и подсказок

### Что делать

- ✅ Большие карточки с воздухом
- ✅ Фото детей как центральный визуальный элемент
- ✅ Прогресс-бары, кольца, мини-графики
- ✅ Микроанимации при действиях (оплата, отметка)
- ✅ Персонализация: «Доброе утро, Анна!»
- ✅ Контекстные подсказки вместо документации

### Голос бренда (Tone of Voice)

| Контекст | Тон | Пример |
|----------|-----|--------|
| Успех | Радостный, энергичный | «Отлично! Абонемент продлён 🎉» |
| Напоминание | Дружелюбный | «Через 3 дня заканчивается абонемент Артёма» |
| Ошибка | Спокойный, помогающий | «Что-то пошло не так. Попробуйте ещё раз» |
| Пустое состояние | Мотивирующий | «Пока нет фото — но скоро тренер загрузит!» |

### Общие требования к Dashboard

1. **Первый экран после входа** — всегда Dashboard, не список или таблица
2. **Персонализация** — приветствие по имени, данные конкретного пользователя
3. **Крупные карточки** — минимум 3-4 информационных блока above the fold
4. **Быстрые действия** — 3-4 кнопки самых частых операций
5. **Живой контент** — фото, новости, события (не только цифры)
6. **Прогресс** — абонемент, посещаемость, статистика визуально
7. **Адаптивность** — на мобильном карточки стекаются вертикально
8. **Skeleton loading** — при загрузке показывать скелетоны карточек
9. **Пустые состояния** — если нет данных, мотивирующая иллюстрация + CTA
