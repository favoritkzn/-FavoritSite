## Соглашения

### Обозначения

| Метка | Значение |
|-------|----------|
| **MVP** | Реализуется в первой версии |
| **Future** | Запланировано после MVP |
| **Гость** | Неавторизованный пользователь |
| **Родитель** | Роль PARENT |
| **Тренер** | Роль COACH |
| **Админ** | Роль ADMIN |

### Layout-оболочки

| Оболочка | Экраны |
|----------|--------|
| `PublicLayout` | Публичный сайт |
| `AuthLayout` | login, register, forgot/reset password |
| `DashboardLayout` | Все кабинеты (sidebar + topbar) |
| `ShopLayout` | Магазин (Future) |

### Стандартные состояния (для всех экранов)

| Состояние | Поведение |
|-----------|-----------|
| **Loading** | Skeleton повторяет структуру блоков. Без полноэкранного спиннера |
| **Empty** | Иллюстрация + заголовок + описание + CTA (если применимо) |
| **Error** | Inline или блок: спокойный тон, кнопка «Повторить» |
| **Success** | Toast (действия) или отдельная страница (оплата, заказ) |

### Стандартные переходы

- Между страницами кабинета: fade + slide up 8px, 300ms
- Между публичными: fade 200ms
- Модалы: scale 0.95→1 + fade, 200ms

## 1. Главная страница сайта

**URL:** `/`  
**Оболочка:** `PublicLayout`  
**Рендеринг:** SSG + ISR (revalidate: 3600)  
**MVP:** ✅

### Цель
Произвести первое впечатление, донести ценность академии, конвертировать гостя в заявку или регистрацию.

### Пользователи
Гость (основной), все роли (при переходе с сайта).

### Данные
- Название академии, город
- Hero-тексты (из CMS / SiteSettings)
- 3–4 преимущества (иконка + заголовок + текст)
- Статистика: кол-во тренеров, детей, лет работы
- 3 последние новости (заголовок, дата, превью)
- 2–3 публичных тренера (фото, имя, должность)
- CTA-блок «Записаться»
- Контакты в footer

### Действия
- Навигация по разделам сайта
- «Записаться на пробное» → `/contacts`
- «Узнать больше» → `/about`
- «Войти» → `/login`
- Клик по новости → `/news/[slug]`
- Клик по тренеру → `/coaches/[id]`

### Расположение блоков
- **Desktop:** max-width 1280px, centered. Hero — 2 колонки (текст 60% + декоративное фото/иллюстрация 40%)
- **Features:** grid 3 колонки
- **Coaches/News:** grid 3 колонки

### Мобильная версия
- Header: logo слева, hamburger справа → drawer-меню
- Hero: 1 колонка, H1 32px, кнопки full-width stack
- Stats: 3 в ряд (компактно) или горизонтальный scroll
- Features: 1 колонка
- Coaches/News: 1 колонка или horizontal scroll

### Состояния

| Состояние | Поведение |
|-----------|-----------|
| Loading | Skeleton: hero (title + 2 lines + 2 buttons), 3 stat boxes, 3 feature cards |
| Empty (новости) | Секция скрыта или «Скоро здесь появятся новости» |
| Error | «Не удалось загрузить страницу» + [Обновить] |
| Success | — (не применимо) |

### Микроанимации
- Hero badge: fade-in + slide down, delay 0ms
- H1: fade-in + slide up, delay 100ms
- CTA-кнопки: stagger fade-in, delay 200ms
- Feature cards: hover translateY(-2px) + shadow
- Stats: count-up при появлении в viewport (Intersection Observer)

### Переходы
- Клик по навигации → плавный переход на страницу
- CTA → `/contacts` с scroll to form

### Быстрые действия
Нет (публичная страница).

### CTA
| Кнопка | Тип | Назначение |
|--------|-----|------------|
| Записаться на пробное | Primary | Главная конверсия → `/contacts` |
| Узнать больше | Secondary | → `/about` |
| Войти | Primary (header) | → `/login` |
| Записаться (banner) | Primary inverse | → `/contacts` |

## 2. Страница авторизации

**URL:** `/login`  
**Оболочка:** `AuthLayout`  
**MVP:** ✅

### Цель
Безопасный и быстрый вход в личный кабинет за минимальное время.

### Пользователи
Родитель, Тренер, Админ (все авторизованные роли).

### Данные
- Логотип академии
- Заголовок «Вход в личный кабинет»
- Поля: email, пароль
- Ссылка «Забыли пароль?»
- Ссылка «Нет аккаунта? Зарегистрироваться»

### Действия
- Ввод email + пароль
- Submit → API `/auth/login`
- Redirect на Dashboard по роли

- Карточка full-width minus 32px padding

- Success → `/parent/dashboard` | `/coach/dashboard` | `/admin/dashboard` (по роли)

Нет.

## 3. Страница регистрации

**URL:** `/register`  
**Оболочка:** `AuthLayout`  
**MVP:** ✅

### Цель
Регистрация родителя в системе для доступа к личному кабинету.

### Пользователи
Гость → становится Родителем (PARENT). Тренеры и админы создаются только админом.

### Данные
- Имя, Фамилия
- Email
- Телефон (опционально)
- Пароль, Подтверждение пароля
- Чекбокс согласия с политикой конфиденциальности

### Действия
- Заполнение формы
- Submit → API `/auth/register`
- Redirect → `/parent/dashboard` (или экран «Ожидайте активации» если PENDING)
- Переход на `/login`

### Расположение блоков
- Карточка 480px max-width (шире login из-за большего числа полей)
- Имя + Фамилия: 2 колонки на desktop, 1 на mobile

### Мобильная версия
- Все поля full-width, stack
- Имя/Фамилия — stack (не 2 колонки)

### Состояния

| Состояние | Поведение |
|-----------|-----------|
| Loading | Кнопка disabled + spinner |
| Empty | — |
| Error | Inline: «Email уже зарегистрирован», «Пароли не совпадают», валидация полей |
| Success | Redirect на dashboard или экран «Аккаунт создан. Администратор активирует доступ» |

### Микроанимации
- Валидация поля: border красный + shake при blur с ошибкой
- Success checkmark при корректном пароле (match indicator)

### Переходы
- → `/parent/dashboard` или pending-screen
- ← `/login`

### CTA
| Кнопка | Тип | Действие |
|--------|-----|----------|
| Создать аккаунт | Primary | Submit (disabled пока нет согласия) |
| Войти | Link | → `/login` |

## 4. Dashboard родителя

**URL:** `/parent/dashboard`  
**Оболочка:** `DashboardLayout`  
**MVP:** ✅ — **ключевой экран продукта**

### Цель
Дать родителю всю важную информацию за 3 секунды. Мотивировать открывать ежедневно.

### Пользователи
Родитель (основной), Админ (при входе в parent-раздел).

### Данные
- Приветствие: «Доброе утро, {имя}!» (по времени суток)
- Выбранный ребёнок (если несколько — tabs/switcher)
- ChildCard: фото, имя, группа, возраст
- Ближайшая тренировка: дата, время, зал, тренер
- Абонемент: прогресс-бар (8/12), дата окончания
- Статистика сезона: голы, передачи, % посещаемости
- 4 превью последних фото
- 1 последняя новость академии
- Счётчик непрочитанных уведомлений

### Действия
- Переключение между детьми
- Quick actions (см. ниже)
- Клик по тренировке → `/parent/schedule`
- Клик по абонементу → `/parent/subscription`
- Клик по фото → `/parent/media`
- Клик по новости → `/news/[slug]`

## 5. Dashboard тренера

**URL:** `/coach/dashboard`  
**Оболочка:** `DashboardLayout`  
**MVP:** ✅

### Цель
Дать тренеру мгновенный обзор дня и доступ к частым действиям за 1 тап.

### Пользователи
Тренер, Админ.

### Данные
- Дата + «Сегодня, {дата} · N тренировок»
- Следующая тренировка: время, группа, зал, кол-во детей
- Список групп тренера (название + кол-во детей)
- Последние 2 объявления
- Статус посещаемости сегодня (отмечена / нет)

### Действия
- «Отметить посещаемость» → `/coach/attendance/[sessionId]`
- Переход к группе → `/coach/groups/[id]`
- Quick actions
