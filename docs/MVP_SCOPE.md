# ФК «Фаворит» — MVP Scope

> Версия: 1.0 | Дата: 30.06.2026  
> Контекст: академия с **40–60 детьми**, **6–7 тренерами**, **1 администратор**  
> Связанные документы: [PRODUCT_VISION.md](./PRODUCT_VISION.md) · [ARCHITECTURE.md](./ARCHITECTURE.md)

---

## Принцип разделения

> **MVP = минимум, чтобы академия полноценно работала без Excel и звонков.**  
> **Future = усиление, автоматизация, масштабирование.**

Если функция не блокирует ежедневную работу академии — она в Future.

## MVP: План разработки (пересмотренный)

### Фаза 1 — Foundation (Недели 1-2)
- Monorepo setup
- Database schema (только MVP-сущности)
- Auth module (JWT, RBAC)
- UI Kit foundation (красная палитра, карточки, скелетоны)
- Docker dev environment

### Фаза 2 — Public Site (Недели 3-4)
- Layouts, главная, о школе, контакты
- Тренеры, расписание, стоимость
- Новости, галерея
- SEO

### Фаза 3 — Admin Core (Недели 5-6)
- Admin Dashboard
- CRUD: дети, родители, тренеры, группы
- Расписание, абонементы

### Фаза 4 — Coach Cabinet (Недели 7-8)
- Coach Dashboard
- Посещаемость (ключевая функция)
- Матчи, голы, передачи
- Фото, объявления

### Фаза 5 — Parent Cabinet (Недели 9-10)
- Parent Dashboard (самый красивый экран)
- Дети, расписание, посещаемость
- Абонементы, оплаты + ЮKassa
- Статистика, уведомления

### Фаза 6 — Polish & Launch (Недели 11-12)
- Email-уведомления
- Empty/error/success states
- E2E тесты критических путей
- Performance optimization
- Production deployment

**Итого MVP: ~12 недель** (вместо 16 за счёт исключения магазина, турниров, сборов, автоматизации)

## MVP: API Endpoints (сокращённый)

В MVP реализуется ~55 endpoints (из ~90 полных). Исключены:
- `/tournaments/*`
- `/camps/*`
- `/shop/*`, `/cart/*`, `/orders/*`
- `/payments/methods/*` (сохранённые карты)
- `/statistics/*` (детальная — только dashboard metrics)
- `/search` (глобальный)
- `/cms/pages/*`

## Критерии готовности MVP

- [ ] Родитель может: войти, увидеть Dashboard, расписание, посещаемость, оплатить абонемент
- [ ] Тренер может: войти, отметить посещаемость, добавить матч с голами, загрузить фото
- [ ] Админ может: управлять детьми, группами, тренерами, расписанием, абонементами
- [ ] Гость может: просмотреть сайт, увидеть тренеров, расписание, новости, записаться
- [ ] Все Dashboard-ы реализованы и являются главным экраном
- [ ] Интерфейс не выглядит как CRM
- [ ] Mobile-friendly для родителей и тренеров
- [ ] Онлайн-оплата через ЮKassa работает
