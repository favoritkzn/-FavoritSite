## 1. Главный принцип

> **Если пользователь чувствует, что работает с CRM — мы проиграли.**

Каждое решение в интерфейсе проверяется одним вопросом: *«Это ощущается как спортивное приложение или как офисная программа?»*

---

## 2. Десять UX-законов «Фаворит»

### 1. Максимальная простота

- Один экран — одна главная задача
- Убрать всё, что не помогает выполнить задачу прямо сейчас
- Если элемент можно убрать без потери смысла — убрать
- Текст кнопок: глагол + объект («Оплатить абонемент», не «Отправить»)

### 2. Минимальное количество кликов

| Задача | Максимум кликов |
|--------|-----------------|
| Увидеть расписание ребёнка | 0 (на Dashboard) |
| Оплатить абонемент | 2 (Dashboard → Оплатить → Подтвердить) |
| Отметить посещаемость группы | 2 (Dashboard → Отметить → Сохранить) |
| Посмотреть фото с тренировки | 1 (Dashboard → Альбом) |
| Добавить результат матча | 3 (Матчи → Новый → Заполнить → Сохранить) |
| Найти ребёнка (админ) | 2 (Поиск → Результат) |

**Правило:** если задача требует > 3 кликов — пересмотреть информационную архитектуру.

### 3. Никаких перегруженных таблиц

| ❌ Не делать | ✅ Делать |
|-------------|----------|
| Таблица на 12 колонок | Карточки с ключевой информацией |
| Строка = 40px высотой | Карточка = 80-120px с воздухом |
| Сортировка по 8 полям | Фильтр по 2-3 ключевым |
| «Показать 50 записей» | Пагинация по 12-20 + infinite scroll |
| Действия в выпадающем ⋮ | Явные кнопки на карточке |

**Исключение:** Admin-раздел «Оплаты» и «Посещаемость» могут иметь компактную таблицу, но:
- Максимум 6 колонок
- Альтернативный card-view toggle
- Sticky header
- Цветовая индикация статусов

### 4. Не CRM — спортивный продукт

| CRM-паттерн | Наш паттерн |
|-------------|-------------|
| Sidebar с 15 пунктами | Sidebar с 5-6 + Dashboard |
| Breadcrumbs на каждой странице | Только при глубине > 2 |
| Модальные формы на 15 полей | Пошаговые wizard или inline-edit |
| Фильтры-аккордеон с 20 полями | 2-3 pill-фильтра |
| «Создать запись» | «+ Добавить ребёнка» |
| Статус-бейдж «ACTIVE» | Зелёная точка + «Активен» |
| Пагинация «1 2 3 ... 47» | «Показать ещё» |
| Export to CSV | Не в MVP |

### 5. Понятная структура каждой страницы

Каждая страница следует единому шаблону:

```
┌─────────────────────────────────────────┐
│  Page Header                            │
│  Заголовок + описание + primary action  │
├─────────────────────────────────────────┤
│  Content Area                           │
│  Карточки / список / детали             │
├─────────────────────────────────────────┤
│  (Optional) Secondary content           │
│  Связанная информация, подсказки        │
└─────────────────────────────────────────┘
```

**Page Header всегда содержит:**
1. Заголовок (что это за страница)
2. Краткое описание или контекст (опционально)
3. Primary action справа (если есть)

### 6. Основные действия за 1–2 клика

**Быстрые действия (Quick Actions)** — обязательный элемент Dashboard и ключевых страниц.

| Роль | Quick Actions |
|------|---------------|
| Родитель | Оплатить, Расписание, Статистика, Уведомления |
| Тренер | Посещаемость, Добавить матч, Загрузить фото |
| Админ | + Ребёнок, Расписание, Оплаты, Статистика |

Quick Actions — это крупные кнопки/карточки, не пункты меню.

### 7. Контекст вместо навигации

Показывать информацию там, где она нужна:

- Абонемент ребёнка — на карточке ребёнка И на Dashboard
- Посещаемость — в карточке ребёнка И на отдельной странице
- Ближайшая тренировка — на Dashboard, не заставлять идти в «Расписание»

### 8. Прогрессивное раскрытие

- Сначала — ключевая информация
- Детали — по клику / expand
- Настройки — на отдельной странице, не перегружают основной flow

Пример: карточка ребёнка показывает имя, группу, фото. Клик → полная статистика, история, документы.

### 9. Обратная связь на каждое действие

| Действие | Обратная связь |
|----------|----------------|
| Клик по кнопке | Hover + loading state |
| Сохранение | Toast «Сохранено ✓» |
| Ошибка | Inline-сообщение + toast |
| Загрузка | Skeleton, не спиннер |
| Удаление | Confirm dialog |
| Долгая операция | Progress bar |

### 10. Mobile-first для тренеров и родителей

- Тренер отмечает посещаемость с телефона на поле
- Родитель смотрит фото с работы
- Все ключевые сценарии должны работать на экране 375px

## 3. Информационная архитектура

### Навигация родителя (5 пунктов + Dashboard)

```
🏠 Dashboard          ← главный экран
👤 Мои дети
📅 Расписание
💳 Оплаты
🔔 Уведомления
⚙️ Настройки
```

Всё остальное (статистика, турниры, сборы, медиа, магазин, документы) — доступно с Dashboard через карточки и quick actions.

### Навигация тренера (5 пунктов + Dashboard)

```
🏠 Dashboard
📅 Расписание
👥 Группы
✓  Посещаемость
⚽ Матчи
⚙️ Настройки
```

### Навигация администратора (7 пунктов + Dashboard)

```
🏠 Dashboard
👶 Дети
👨‍👩‍👧 Родители
🏃 Тренеры
📅 Расписание
💰 Финансы
⚙️ Настройки
```

Магазин, турниры, сборы, медиа, новости — подразделы, доступные из Dashboard или соответствующих разделов.

## 4. Паттерны взаимодействия

### Карточка вместо строки

```
❌ CRM-строка:
| Иванов Артём | 2015 | Группа А | Активен | 12/12 | ✏️ 🗑️ |

✅ Наша карточка:
┌────────────────────────────┐
│ [Фото]  Артём Иванов       │
│         Группа 2015         │
│         ████████░░ 8/12     │
│         [Подробнее]         │
└────────────────────────────┘
```

### Inline-edit вместо модалки

Для простых полей (имя, телефон, email) — редактирование на месте, не модальное окно.

### Wizard вместо длинной формы

Создание ребёнка (админ):
1. Основное (имя, дата рождения)
2. Группа (выбор группы)
3. Родитель (привязка)
4. Готово (подтверждение)

### Подтверждение только для деструктивных действий

- Удаление — confirm dialog
- Сохранение, создание, оплата — без confirm (только success feedback)

## 5. Доступность (a11y)

- Контраст текста: минимум 4.5:1 (WCAG AA)
- Focus-visible на всех интерактивных элементах
- Alt-текст для всех изображений
- Keyboard navigation для форм и модалов
- `aria-label` для icon-only кнопок
- Не полагаться только на цвет (статус = цвет + иконка + текст)

## 6. Копирайтинг (UX Writing)

| Контекст | ❌ | ✅ |
|----------|----|----|
| Пустой список | «Нет данных» | «Пока нет фото с тренировок» |
| Кнопка | «Submit» | «Сохранить» |
| Ошибка | «Error 500» | «Не удалось загрузить. Попробуйте ещё раз» |
| Загрузка | «Loading...» | Skeleton (без текста) |
| Удаление | «Delete record?» | «Удалить ребёнка из группы?» |
| Успех | «Success» | «Посещаемость сохранена» |

---

## 7. Антипаттерны (чего избегать)

1. **Hamburger menu на desktop** — sidebar всегда виден
2. **Модальное окно внутри модального окна**
3. **Более 2 уровней вложенности навигации**
4. **Автоматический redirect без объяснения**
5. **Форма без кнопки «Назад»**
6. **Более 5 полей в одной форме без wizard**
7. **Обязательные поля без визуальной маркировки**
8. **Disabled кнопка без объяснения почему**
9. **Бесконечная прокрутка без индикатора**
10. **Таблица как единственный способ просмотра данных**
